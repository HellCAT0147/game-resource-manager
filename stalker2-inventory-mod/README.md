# LootHelper — мод-мост «инвентарь STALKER 2 → выгодность»

Читает инвентарь игрока прямо в игре (через UE4SS + Lua) и потом либо рисует
оверлей «что нести / что продать» поверх игры, либо выгружает `inventory.json`
для веб-приложения. Игра у нас **не обновляется** → настраиваем один раз, работает всегда.

> **Сейчас здесь только ШАГ 1 — разведка.** Скрипт пока НЕ читает инвентарь
> по-настоящему: он проверяет, что UE4SS работает, дотягивается до игрока и
> печатает имена классов. По ним мы найдём объект инвентаря и его поля
> (стоимость/вес/кол-во), а уже потом я напишу настоящий дампер/оверлей.

Исходник мода живёт **в этом гит-репозитории** (`stalker2-inventory-mod/`).
В папку игры кладётся только копия — так всё версионируется здесь, а не теряется
в игровых файлах. См. раздел «Git и папка игры» ниже.

## Шаг 0. Поставить UE4SS (один раз; игра v1.9, build 89790)

> Под 1.9 НЕ бери версионные старые пакеты («для 1.7» и т.п.) — нужен свежий UE4SS.
> Он на активной разработке и обновляется чаще игры.

1. Выбери один из вариантов:
   - **Проще всего — авто-апдейтер:** [UE4SS-Bootstrap (mods/2285)](https://www.nexusmods.com/stalker2heartofchornobyl/mods/2285) — кидаешь один файл, он сам скачивает и держит рабочую версию UE4SS. Версию угадывать не надо.
   - **Или STALKER2-пакет:** [UE4SS updated (mods/1910)](https://www.nexusmods.com/stalker2heartofchornobyl/mods/1910) — уже с нужными сигнатурами; в комментах глянь, что подтверждают на 1.9.
   - **Для продвинутых:** свежий experimental с [GitHub UE4SS releases](https://github.com/UE4SS-RE/RE-UE4SS/releases) (может понадобиться подложить STALKER2-сигнатуры).
2. Распакуй/скопируй в папку с игровым exe:
   ```
   d:\Games\STALKER2\Stalker2\Binaries\Win64\      (рядом со Stalker2-Win64-Shipping.exe)
   ```
   Появится прокси-DLL (`dwmapi.dll`) и папка UE4SS (`ue4ss\` или сразу `Mods\`).
3. Запусти игру — должно всплыть **отдельное окно консоли UE4SS**, и появиться `...\ue4ss\UE4SS.log`. Всплыло → UE4SS жив.
   (Не стартует/чёрный экран — удали `dwmapi.dll` (мгновенно выключает UE4SS) и возьми другой вариант из п.1.)

## Шаг 1. Поставить этот мод и снять разведданные

1. Найди папку `Mods` своей установки UE4SS — там, где лежит `mods.txt` и
   примеры вроде `BPModLoaderMod`. Обычно это:
   ```
   ...\Binaries\Win64\ue4ss\Mods\        (новый UE4SS)
   ...\Binaries\Win64\Mods\              (старый UE4SS)
   ```
2. Скопируй туда папку **`LootHelper`** целиком (из этого репо).
   Файл `enabled.txt` внутри уже включает мод.
3. Запусти игру, **загрузи сейв**, открой инвентарь и нажми **F8**.
4. Забери лог: `...\Binaries\Win64\**\UE4SS.log` (или прямо из окна консоли).
   Пришли мне строки, начинающиеся с `[LootHelper]`.

## Шаг 1б. Live View (главный инструмент разведки)

В окне UE4SS есть GUI с вкладкой **Live View** — это интерактивный обозреватель
объектов игры:

1. В окне UE4SS открой вкладку **Live View**.
2. В поиске введи имя класса Pawn (его напечатает F8-скан, строка `Pawn класс:`).
3. Разверни найденный экземпляр и поищи среди полей/компонентов что-то со словом
   **Inventory / Items / Container**.
4. Разверни инвентарь → внутри должен быть массив предметов; у одного предмета
   найди поля вроде **Cost / Price / Weight / Count / Amount** и его **Name/ID**.
5. Сфоткай/выпиши эти имена (класс инвентаря, имя поля-массива, имена полей
   предмета) и пришли мне. Этого хватит, чтобы написать настоящий дампер.

## Git и папка игры

Всю игру под git заворачивать **не надо**: там десятки гигабайт бинарных `.pak`
(один `pakchunk0` — ~26 ГБ), git ими подавится, а трогаем мы только крошечную
папку мода. Поэтому:

- **Исходник мода** версионируется здесь, в этом репозитории.
- В игру попадает лишь копия `LootHelper`.
- Удобно для итераций: вместо копирования сделать **junction** (симлинк папки),
  чтобы правки в репо сразу были видны игре:
  ```cmd
  mklink /J "d:\Games\STALKER2\Stalker2\Binaries\Win64\ue4ss\Mods\LootHelper" "d:\Documents\Git\subnautica\subnautica 2\resource manager\stalker2-inventory-mod\LootHelper"
  ```
  (путь к `Mods` подставь свой; команда — в cmd от админа.)
